## Key Findings

### 1. High UNVERIFIABLE Rate (60.9%)

The analysis claims (C-100 to C-145) are heavily weighted toward external tool
evaluations and cross-locale architecture recommendations. 28 of 46 claims
reference tools not installed on this machine or environments not accessible
from here. This is inherent to the research scope (evaluating external options)
but means the majority of claims rest on secondary source analysis rather than
ground-truth verification.

### 2. Numeric Errors in Verifiable Claims

Where claims COULD be verified, several contained material numeric errors:

| Claim | Stated                        | Actual                                             |
| ----- | ----------------------------- | -------------------------------------------------- |
| C-102 | canonical-memory has 20 files | **25 files**                                       |

| C-115 | 25 hooks                      | **26 hooks**                                       |
| C-119 | CLAUDE.md ~135 lines          | **258 lines** (CLAUDE.md claims 135 in its header) |
| C-127 | 39 individual memory files    | **43 files**                                       |
| C-138 | 39 files                      | **43 files**                                       |

### 3. One Clear Refutation

C-141 (Engram has "no Windows compatibility issues documented") is directly
contradicted by Gap G1/G7 findings showing antivirus false positives,
path-with-spaces bugs, SQLITE_BUSY concurrency, and silent data loss. The
original 4/5 score was correctly downgraded to 2/5.

### 4. Strongest Verified Claims

The following claims were fully confirmed against the codebase:

- **C-103**: Path-key problem is real (different usernames = different memory
  directories)
- **C-114**: Hook-based capture is the only automatic mechanism (26 hooks
  confirmed)
- **C-116**: mcp\_\_memory is configured but underused
- **C-121**: Episodic memory is the weakest layer
- **C-122**: MEMORY.md index pattern is partially implemented
- **C-123**: AutoDream shows zero observable activity
- **C-128**: Learning consolidation exists (every 10 reviews) but session memory
  has no equivalent
- **C-132**: Multi-layer system already exists implicitly
- **C-135**: autoMemoryDirectory at git-tracked directory is
  untested/unconfigured

### 5. autoMemoryDirectory Verification

The central architectural recommendation (point autoMemoryDirectory at
canonical-memory) has been confirmed as:

- **Not configured** (zero matches in any settings file)
- **Architecturally untested** (no community implementations found)

  session)
