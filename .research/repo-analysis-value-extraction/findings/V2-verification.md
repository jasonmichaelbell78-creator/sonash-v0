## Summary: HIGH-Impact Issues Found

### Issue 1: C-AI-13 — Precision Error on Linux Foundation Donation (PARTIAL REFUTE)

**What the claim says:** "Anthropic donated Agent Skills specification to Linux
Foundation December 2025."

**What the source actually supports:** It was **MCP** (Model Context Protocol)
that was donated to the Agentic AI Foundation under the Linux Foundation in
December 2025. The Agent Skills specification was "released as an open standard"
— but the Linux Foundation donation attribution is for MCP, not Agent Skills.

**Skill design impact:** Low. The ecosystem scale numbers are accurate. The
Agent Skills going cross-platform (OpenAI adoption) is still supported at MEDIUM
confidence. Recommend correcting the research output to read: "Anthropic donated
MCP to the Linux Foundation in December 2025. The Agent Skills specification was
released as an open standard; OpenAI adopted the same format for Codex CLI."

---

### Issue 2: C-AI-01 — 20-50% Accuracy Improvement Is Practitioner-Sourced, Not Peer-Reviewed

**What the claim says:** Structured analysis beats open-ended with "20-50%
accuracy improvement cited."

**What the source actually supports:** D4a explicitly notes in its Gaps section:
"No controlled studies specifically comparing structured vs. open-ended for
codebase analysis output quality." The 20-50% figure comes from practitioner
sources.

**Skill design impact:** Low directionally (structured is still recommended),
but the specific range should not be cited as empirical. Recommend downgrading
the percentage claim to MEDIUM confidence or dropping the specific range.

---

### Issue 3: C-AI-08 — 17-Point Accuracy Decline Is Single-Source, Blog Post

**What the claim says:** "17-point retrieval accuracy decline (93% to 76%)
measured as context fills on Claude Opus 4.6 at 1M tokens."

**What the source actually supports:** This comes from anthropic.com/engineering
blog post (Source 5 in D4b), not a peer-reviewed study. The Augment-vs-1M
comparison supporting "200K beats 1M" is separately confirmed. The 93%→76%
specific numbers have only one source at MEDIUM trust.

**Skill design impact:** Low. The directional finding (large context degrades
quality) is multiply-confirmed. Only the specific 17-point decline is
single-sourced. The design implication (use subagents, avoid naively loading
large repos) is correct regardless.

---

### Issue 4: C-AI-18 — CVE/Malicious Skill Counts Are Single-Source

**What the claim says:** "24 CVEs identified in Claude Code ecosystem; 655
malicious skills detected."

**What the source actually supports:** Single source — a community audit, no
independent cross-reference. The numbers are plausible and the directional claim
(external skill supply chain risk) is confirmed by the ecosystem research, but
the specific counts carry uncertainty.

**Skill design impact:** Low. The security argument for SoNash's internal-only
skill system stands regardless of exact counts.
