SWS is a META-PLAN — it plans the sequence, gates, and coordination for building
out other plans.

**Why:** User corrected emphatically when AI tried to "extract items" from child
plans into SWS. Child plans (Tooling, CQ, DE) are execution appendices.
DECISIONS-reeval.md defines how they connect/sequence. SWS does NOT absorb their
internal content.

**How to apply:** SWS references child plans + defines gates between them. Child
PLAN.md files are their own execution docs. DECISIONS-reeval.md Q# decisions
define integration points. Never pull child plan content into SWS.
