## Purpose

Snapshot of the learning-system signal: violations-per-PR, recurring-category
ratio, and week-over-week trend. Auto-regenerated by the SessionStart
learning-effectiveness analyzer; manual edits will be overwritten on next run.

Use this file to answer "is the project actually learning from documented
patterns?" Read alongside `docs/AI_REVIEW_LEARNINGS_LOG.md` (qualitative
learnings per review) and `.claude/state/hook-warnings-log.jsonl` (raw signal).

## AI Instructions

status), the Purpose / AI Instructions / Quick Start / Version History sections,
and replace only the metrics table below. Bump the **Last Updated** date to the
regeneration date. Bump **Document Version** only when the schema or sections
change — not for routine metric refreshes.

## Quick Start

- **Inspect raw signal:** `.claude/state/hook-warnings-log.jsonl`
- **Regenerate manually:** SessionStart hook does this automatically; trigger
  with `node .claude/hooks/session-start.js` or open a new session.
- **Investigate a metric:** Cross-reference recurring categories against
  `docs/agent_docs/CODE_PATTERNS.md` to find the documented pattern that is not
  converging.
