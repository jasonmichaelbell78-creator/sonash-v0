## Architecture & Orchestration

| #   | Decision            | Choice                                                                                                                                                                                                                                                                                                                                                                                                                                                         | Rationale                                                                                                                                               |
| --- | ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 4   | Determination phase | 4-level model based on structural complexity. L1-L3 use skill-as-orchestrator, L4 uses agent team when sub-questions are interdependent. Boundary is coordination need (do agents need to react to each other's findings?).                                                                                                                                                                                                                                    | Structural complexity drives orchestration model, not arbitrary thresholds. Refined from initial 5-level proposal — L4 (Deep Investigation) absorbs L5. |
| 6   | Searcher approach   | Hybrid — default to specialists (web, codebase, community, security, academic), orchestrator decides composition based on question classification                                                                                                                                                                                                                                                                                                              | Specialized variants produce qualitatively different and complementary findings (proven by Supabase-vs-Firebase example)                                |
| 7   | Agent allocation    | Formula: `D + 3 + floor(D/5)` where D = number of research dimensions. Dimension-driven, not capped. Budget is a guardrail, not a constraint.                                                                                                                                                                                                                                                                                                                  | Validated against 23-agent empirical run (formula predicts 24, actual was 23)                                                                           |
| 14  | Agent role map      | 9 roles total. 2 custom agents (searcher forked from gsd-project-researcher, synthesizer forked from gsd-research-synthesizer). 2 convergence-loop presets (contrarian, outside-the-box). 1 role subsumed by inline self-audit (auditor — implemented in Phase 4 self-audit rather than as a separate CL preset). 4 conditional roles (source verifier, cross-pollinator, domain specialist, scout — implemented as presets/modes/configs, not custom agents). | Minimizes custom agent maintenance while covering all research functions                                                                                |
| 11  | All agent tools     | Every role gets Read + Write minimum. Searcher: +Bash, Grep, Glob, WebSearch, WebFetch, Context7. Synthesizer: +Bash. Contrarian/OTB: +Bash, WebSearch, WebFetch.                                                                                                                                                                                                                                                                                              | All agents must be able to read input and write output                                                                                                  |
| 20  | Resume behavior     | Both automatic (session-start detects incomplete research) AND manual (re-invoke same topic detects state file). Belt and suspenders.                                                                                                                                                                                                                                                                                                                          | Proven pattern from deep-plan compaction resilience                                                                                                     |

## Depth & Rigor

| #   | Decision                     | Choice                                                                                                                                                                                          | Rationale                                   |
| --- | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------- |
| 5   | Depth levels                 | Quick/Standard/Deep ELIMINATED. Floor is L1 (Exhaustive). Scale: L1 → L2 → L3 → L4. If someone wants quick, they ask Claude directly — `/deep-research` does what a single conversation cannot. | User directive: "exhaustive is the minimum" |
| 3   | Contrarian + outside-the-box | Mandatory at ALL levels. No exceptions.                                                                                                                                                         | User directive: overkill on everything      |
| 15  | Contrarian/OTB scaling       | L1-L2: 1 agent each (CL presets). L3: 2 agents each (different adversarial/lateral strategies). L4: 3 agents each + red team pass + pre-mortem.                                                 | Scales effort with structural complexity    |
| 8   | Budget                       | NOT a design consideration. User adjusts if it becomes an issue. Do not optimize for cost at the expense of thoroughness.                                                                       | User directive                              |

| --- | ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |

## Core Design Principle

> **Overkill by default. Leave no stone unturned. Go above, beyond, and far past
> what seems necessary or appropriate.**

This is not a suggestion — it is the foundational design principle that
overrides efficiency-first assumptions throughout the entire system.
