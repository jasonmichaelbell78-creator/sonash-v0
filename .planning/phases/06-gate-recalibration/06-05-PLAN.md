<task type="auto">
  <name>Task 1: Add --auto mode to archive-reviews.js and temporal gap detection to check-review-archive.js</name>
  <files>scripts/archive-reviews.js, scripts/check-review-archive.js</files>
  <action>
  **archive-reviews.js changes (GATE-04):**

1. Add `--auto` flag parsing alongside existing `--apply`.
2. When `--auto` is passed: skip the confirmation prompt and behave like
   `--apply` (execute archival directly).
3. Add a safety check: if the script detects it's running during an active
   session (check for `.claude/hooks/.session-active` or similar indicator), log
   a warning and exit without archiving. This prevents mid-session data loss per
   Pitfall 6 in research.
4. If `--auto` and threshold is NOT exceeded, exit silently (no output, exit 0).
5. Add `--keep N` flag (if not already present) to specify how many recent
   reviews to keep (default: current threshold value).

**check-review-archive.js changes (GATE-09):**

1. Add a new Section 5: "Temporal Coverage" after the existing sections.
2. Read all reviews from reviews.jsonl that have a `date` field.
3. Group reviews by ISO week (YYYY-WNN format).
4. Detect gaps: any week with zero reviews where both the prior and following
   weeks have reviews.
5. Detect long gaps: any stretch of 2+ consecutive weeks with no reviews.
6. Output temporal gaps in the existing report format:
   ```
   Section 5: Temporal Coverage
   ============================
   Weeks with reviews: 42
   Weeks with gaps: 3
   Longest gap: 2 weeks (2025-W48 to 2025-W49)
   Gap details:
     2025-W48: No reviews (between W47: 3 reviews and W50: 2 reviews)
   ```
7. Export the temporal analysis function with `require.main === module` guard
   for testing. </action> <verify>Run
   `node scripts/archive-reviews.js --auto --dry-run` (if dry-run mode exists)
   or `node scripts/archive-reviews.js --help` to confirm --auto is recognized.
   Run `node scripts/check-review-archive.js` and verify Section 5 appears in
   output.</verify> <done>archive-reviews.js supports --auto mode with session
   safety check; check-review-archive.js surfaces temporal coverage gaps by
   week</done> </task>
