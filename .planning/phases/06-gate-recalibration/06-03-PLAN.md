   `defer_count >= 2 AND status === "open" AND promoted_to_debt === false`.

3. For each item to escalate: a. Call `scripts/debt/intake-pr-deferred.js` via
   `execFileSync('node', [...])` with:

   - `--file` set to `"deferred-escalation"`
   - `--title` set to the item's `finding` field
   - `--severity S1`
   - `--category` set to the item's `category` field (default "code-quality")
   - `--description` "Auto-escalated: deferred N times without resolution" b.

     `promoted_to_debt: true`. c. Rewrite deferred-items.jsonl with the updated
     items (full file rewrite -- file is small, <100 records; use
     `safeWriteFile` from `scripts/lib/security-helpers.js`).

   - `--dry-run`: show what would be escalated without modifying anything
   - `--threshold N`: override default defer_count threshold (default 2)

   escalation".

   can be imported by tests.
