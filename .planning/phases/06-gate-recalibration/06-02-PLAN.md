<task type="auto">
  <name>Task 1: Add --analytics mode to log-override.js</name>
  <files>scripts/log-override.js</files>
  <action>
  Add analytics subcommand to the existing log-override.js CLI:

1. Parse new flags: `--analytics`, `--days N` (default 30), `--json`
   (machine-readable output).

2. When `--analytics` is passed, read override-log.jsonl and compute:
   - `period`: { days, from, to }
   - `total`: total overrides in period
   - `byCheck`: object mapping check name to count, sorted descending
   - `byBranch`: object mapping branch to count (top 10)
   - `noReasonCount`: entries where reason is empty, "No reason", or under 10
     chars
   - `noReasonPct`: percentage of no-reason entries
   - `patterns`: array of repeat overrides (same check overridden 3+ times on
     same branch) with { branch, check, count }
   - `trend`: { current_week, previous_week, change_pct } comparing last 7 days
     vs prior 7 days

3. **Path sanitization:** When displaying branches or any output, never include
   absolute filesystem paths. The `cwd` field in override entries must be masked
   or omitted in analytics output. Use `sanitizeError` pattern from
   `scripts/lib/sanitize-error.js` for any error messages.

4. Human-readable output format (default):

   ```
   Override Analytics (last 30 days)
   ==================================
   Total overrides: 83
   No-reason rate: 16.3% (27/166)

   By Check:
     cross-doc-deps    43  (51.8%)
     pattern-compliance 15  (18.1%)
     ...

   Patterns (3+ same check on same branch):
     branch/feature-x  cross-doc-deps  5 overrides
     ...

   Trend: -12% vs previous week
   ```

5. JSON output (when `--json`): print the analytics object as JSON to stdout.

6. Use existing JSONL reading logic already in log-override.js (the `--list`
   mode reads the file). Reuse that file reading, add filtering by date.
   </action> <verify>Run `node scripts/log-override.js --analytics --days 30`
   and confirm it produces output without errors. Run with `--json` and pipe
   through
   `node -e "JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'))"` to
   confirm valid JSON.</verify> <done>--analytics mode produces aggregated
   override data in human-readable and JSON formats, with path
   sanitization</done> </task>
