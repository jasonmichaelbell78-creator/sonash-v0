    **run-health-check.js** — Main runner script. CLI entry point:
    ```
    node scripts/health/run-health-check.js [--quick|--full] [--json] [--dimension=ID]
    ```

    Behavior:
    - `--quick`: Run only fast checkers (code-quality, test-coverage, debt-health, security) for ~5s execution. Skip slow external checks (GitHub Actions, SonarCloud, review sync).
    - `--full` (default): Run all 10 checkers.
    - `--json`: Output raw JSON to stdout (for programmatic consumption).
    - `--dimension=ID`: Drill into a specific dimension, showing all metrics with benchmarks, scores, and trend.
    - Default (no flags): Run all checkers, output formatted text summary to stdout with category grades and composite score.

    Implementation:
    1. Import all 10 checkers from ./checkers/
    2. Run them (full or quick subset)
    3. Pass results to computeCompositeScore from ./lib/composite.js
    4. If --dimension, call getDimensionDetail from ./lib/dimensions.js
    5. Format output (JSON or text table with grades)
    6. Output to stdout, progress to stderr

    Text output format (when no --json):
    ```
    Ecosystem Health Report
    =======================
    Composite: B (82/100)

    Category Scores:
      Code Quality      A  (92)
      Security          B  (85)
      Technical Debt    C  (72)
      ...

    Dimensions with issues:
      debt-aging         C  (70) - 3 items > 30 days old
      hook-pipeline      D  (62) - 5 warnings in last 7 days
    ```
