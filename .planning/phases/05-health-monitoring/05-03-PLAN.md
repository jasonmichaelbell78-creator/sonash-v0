    **1. scripts/health/lib/mid-session-alerts.js** — Lightweight mid-session checks:

    Export `runMidSessionChecks()` that performs fast, targeted degradation detection:

    a. **Deferred item aging** — Read data/ecosystem-v2/deferred-items.jsonl, find items with lifecycle "open" or "in-progress" older than 30 days. If any found, create a warning via createWarning from warning-lifecycle.js and surface via append-hook-warning.js.

    b. **Duplicate detection** — Read data/ecosystem-v2/deferred-items.jsonl, check for items with identical title+category added in the last 7 days. If duplicates found, alert.

    c. **Score degradation** — Read last 2 entries from ecosystem-health-log.jsonl. If score dropped by 10+ points, alert.

    Each check:
    - Uses try/catch for all file reads
    - Calls `createWarning()` from warning-lifecycle.js to persist the finding
    - Calls append-hook-warning.js (via execFileSync) to surface in hook warnings
    - Respects cooldown: check `.claude/hooks/.alerts-cooldown.json` before firing. Use same cooldown pattern — if last alert of same type was < 1 hour ago, skip.

    The function returns { alerts: [ { type, message, severity } ], skipped: number }.

    **Design for event-based trigger:** This script is meant to be called from post-commit hook or on-demand. Keep execution under 2 seconds — no npm commands, no type-checking, just JSONL reads and date comparisons.

    **2. scripts/health/mid-session-alerts.test.js** — Tests using node:test:

    4. Detects score degradation (10+ point drop)
    5. Does not alert for stable/improving scores
    6. Respects cooldown — same alert type within 1 hour is skipped
    7. Returns empty alerts array when all checks pass
