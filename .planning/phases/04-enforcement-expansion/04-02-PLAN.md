  <action>
    Create 7 new ESLint rules following the existing project pattern (see no-shell-injection.js for structure). Use shared utilities from `lib/ast-utils.js` (getCalleeName, unwrapNode, hasStringInterpolation, getEnclosingScope).

    **Rule 1: no-effect-missing-cleanup.js**
    Detect useEffect callbacks that create setInterval/setTimeout but return no cleanup function.
    - Visitor: CallExpression where callee is "useEffect"
    - Check: callback body contains setInterval/setTimeout but no ReturnStatement with argument
    - Message: "useEffect sets interval/timeout but returns no cleanup function"

    **Rule 2: no-unsafe-spread.js**
    Detect spreading unknown/untyped objects into JSX props or function arguments where it could override safety-critical props.
    - Visitor: SpreadElement in JSXOpeningElement
    - Check: spread argument is not from a known-safe pattern (destructured props, typed literal)
    - Message: "Unsafe spread may override critical props (onClick, href, action)"

    **Rule 3: no-state-update-in-render.js**
    Detect setState/dispatch calls in component render body (outside useEffect/handlers).
    - Visitor: CallExpression matching setState pattern
    - Check: not inside useEffect callback, event handler, or useCallback
    - Message: "State update during render causes infinite re-render loop"

    **Rule 4: no-async-component.js**
    Detect async function components (React components cannot be async in client-side code).
    - Visitor: FunctionDeclaration/ArrowFunctionExpression that is async AND returns JSX
    - Check: file is not in app/ server component paths
    - Message: "Client components cannot be async functions"

    **Rule 5: no-missing-error-boundary.js**
    Detect Suspense usage without ErrorBoundary wrapper.
    - Visitor: JSXElement with name "Suspense"
    - Check: no ancestor JSXElement named "ErrorBoundary"
    - Message: "Suspense should be wrapped in ErrorBoundary for error handling"

    **Rule 6: no-unbounded-array-in-state.js**
    Detect useState initialized with array that grows via spread/concat in effect without size limit.
    - Visitor: CallExpression where callee is "useState" with array initializer
    - Track: corresponding setState calls using spread [...prev, newItem] without .slice()
    - Message: "Unbounded array growth in state - add size limit with .slice()"

    **Rule 7: no-callback-in-effect-dep.js**
    Detect useEffect with inline function in dependency array (should use useCallback).
    - Visitor: CallExpression where callee is "useEffect"
    - Check: second argument (deps array) contains function expressions
    - Message: "Inline function in useEffect deps causes re-run every render - use useCallback"

    Register ALL 7 new rules in `eslint-plugin-sonash/index.js` following existing pattern. Each rule added to the `rules` object with appropriate key name.

    Each rule must follow existing conventions:
    - `"use strict";` at top
    - meta.type: "problem" or "suggestion"
    - meta.docs.description and recommended
    - meta.messages object with messageId
    - Use getCalleeName() from ast-utils.js where applicable
