  <action>
    Add 10-15 new rules to the ANTI_PATTERNS array in check-pattern-compliance.js. Follow existing rule structure (id, severity, pattern/testFn, message, fix, review, fileTypes, pathFilter).

    **New rules to add (categorized by pattern type):**

    **Banned imports (4 rules):**
    1. `banned-direct-firestore-write` (critical): Import of setDoc/addDoc/updateDoc/deleteDoc from firebase/firestore in app code. pathExclude: functions/ directory.
    2. `banned-moment-import` (important): Import of 'moment' library (use date-fns or Intl). Lighter alternative exists.
    3. `banned-lodash-full-import` (important): `import _ from 'lodash'` or `require('lodash')` -- use lodash-es or specific imports like `lodash/get`.
    4. `banned-fs-in-client` (critical): Import of 'fs' or 'node:fs' in app/, components/, pages/ directories (server-only module).

    **Naming violations (3 rules):**
    5. `no-generic-handler-name` (important): Functions named `handleClick`, `handleChange`, `handleSubmit` without a descriptive prefix. Pattern: `/\bfunction\s+handle(?:Click|Change|Submit)\b/` -- should be `handleLoginSubmit`, `handleEmailChange`, etc.
    6. `no-single-letter-variable` (edge): Single-letter variable declarations (except i/j/k in loops, e in catch, _ for unused). Use testFn for context analysis.
    7. `no-todo-without-ticket` (important): TODO/FIXME comments without a reference (issue number, ticket ID). Pattern: `// TODO(?!.*(?:#\d|JIRA|GH-|issue))`.

    **Security/safety (3-4 rules):**
    8. `no-process-env-inline` (important): Direct process.env.VARIABLE_NAME access in component/page files. Should use centralized config.
    9. `no-string-concat-in-query` (critical): String concatenation/template literal in database query strings (SQL injection risk). Use testFn for multi-line context.
    10. `no-document-cookie-access` (important): Direct document.cookie read/write (use cookie utility).
    11. `no-window-location-assign` (important): Direct window.location.href assignment (use Next.js router).

    **Correctness (2 rules):**
    12. `no-json-parse-without-try` (important): JSON.parse() call not inside try/catch block. Use testFn for context analysis.
    13. `no-array-index-as-key` (edge): Using array index as React key in .map() -- pattern: `.map((item, index)` followed by `key={index}` or `key={i}`.

    **All rules must include:**
    - Appropriate severity tier (critical/high/medium/low matching existing tiers)
    - fileTypes array scoping to relevant file extensions
    - pathFilter regex to target relevant directories
    - pathExclude where needed (e.g., test files, functions/ dir)
    - All regex patterns must use /g flag (per CLAUDE.md anti-pattern #5)
    - Clear message and fix instructions

    **FP auto-disable feature (ENFR-07):**
    Add logic near the rule-loading section that:
    1. Reads verified-patterns.json exclusion counts per rule
    2. If a rule has more than FP_THRESHOLD (default: 25) exclusions, skip it and log: `[FP-DISABLED] Rule ${id} skipped (${count} exclusions > threshold ${FP_THRESHOLD})`
    3. FP_THRESHOLD configurable via `--fp-threshold=N` CLI flag
    4. When `--fp-report` is used, show disabled rules in a "AUTO-DISABLED" section
    5. Allow override: `--include-fp-disabled` flag forces all rules to run
