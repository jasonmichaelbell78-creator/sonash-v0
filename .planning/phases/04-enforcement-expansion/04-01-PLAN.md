  <name>Task 1: Create Semgrep rule directory structure and 20+ YAML rules</name>
  <files>

  <action>
    Create `.semgrep/rules/` directory with three subdirectories: security/, correctness/, style/.

    **Security rules (8-10 rules, severity: ERROR):**
    - `no-unsanitized-error-response.yml`: Detect raw error.message/stack in response objects. Use pattern-not-inside for sanitizeError() wrapper.
    - `taint-user-input-to-exec.yml`: mode:taint from req.body/query/params/argv to execSync/exec/spawn. Sanitizers: escapeShellArg, validateInput.
    - `taint-path-traversal.yml`: mode:taint from user input to readFileSync/writeFileSync/path.join. Sanitizers: validatePathInDir, isPathContained.
    - `no-direct-firestore-write.yml`: Detect import of setDoc/addDoc/updateDoc/deleteDoc from firebase/firestore in app code (exclude functions/ dir).
    - `no-hardcoded-secrets.yml`: Detect string literals matching API key / token / password patterns assigned to variables.
    - `no-eval-usage.yml`: Detect eval(), Function(), new Function() usage.
    - `no-innerhtml-assignment.yml`: Detect direct innerHTML/outerHTML assignment.
    - `no-dangerouslysetinnerhtml.yml`: Detect dangerouslySetInnerHTML in JSX without sanitization.

    **Correctness rules (7-8 rules, severity: WARNING):**
    - `async-without-try-catch.yml`: Async functions with await but no try/catch. Use pattern-not-inside for existing error boundaries.
    - `file-read-without-try-catch.yml`: readFileSync/readFile calls not inside try/catch.
    - `no-floating-promise.yml`: Async function call without await/then/catch.
    - `regex-without-lastindex-reset.yml`: /g regex used in loop without lastIndex = 0 reset.
    - `no-race-condition-file-ops.yml`: existsSync followed by readFileSync/writeFileSync without try/catch wrapping the read/write.
    - `no-await-in-loop.yml`: await inside for/while loop (suggest Promise.all).
    - `no-unchecked-array-access.yml`: Array index access without length check in critical paths.

    **Style rules (4-5 rules, severity: INFO):**
    - `no-console-in-components.yml`: console.log/warn/error inside React component files (app/, components/ dirs).
    - `no-any-type.yml`: TypeScript `: any` type annotations (exclude test files).
    - `no-inline-firestore-query.yml`: Direct getDoc/getDocs calls outside lib/firestore-service.ts.
    - `no-magic-numbers.yml`: Numeric literals > 1 outside const declarations (critical paths only).

    Each rule MUST include:
    - `id` in format `sonash.{category}.{rule-name}`
    - `languages: [javascript, typescript]`
    - `severity` appropriate to category
    - `message` with clear explanation and fix suggestion
    - `metadata.category` and `metadata.code-pattern-ref` linking to CODE_PATTERNS.md pattern name
    - `pattern-not-inside` or `pattern-not` for known-safe patterns to minimize FP
    - `paths.include` or `paths.exclude` to scope rules to relevant directories

    Create `.semgrep/.semgrepignore` excluding:
    - node_modules/
    - .next/
    - dist/
    - coverage/
    - tests/semgrep/ (test fixture files)
    - docs/
    - *.md
    - *.json
    - *.jsonl

  </action>
