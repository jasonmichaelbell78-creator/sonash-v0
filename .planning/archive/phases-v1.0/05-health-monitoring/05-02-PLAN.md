  <name>Task 1: Warning lifecycle library</name>
  <files>
    scripts/health/lib/warning-lifecycle.js
  </files>
  <action>
    Create `scripts/health/lib/warning-lifecycle.js` — a plain JS module (matching codebase convention for scripts/) that manages warning lifecycle in data/ecosystem-v2/warnings.jsonl.

    **Exported functions:**

    1. `createWarning({ category, message, severity, source_script, related_ids })` — Creates a new warning record:
       - Generates ID: `warn-{Date.now()}` (same pattern as invocation IDs in 03-02)
       - Sets lifecycle: "new", date: today (YYYY-MM-DD), schema_version: 1
       - Sets completeness: "full", completeness_missing: []
       - Sets origin: { type: "manual", tool: source_script || "health-check" }
       - Writes to data/ecosystem-v2/warnings.jsonl using the compiled appendRecord from scripts/reviews/dist/
       - Returns the created record

    2. `acknowledgeWarning(warningId)` — Reads warnings.jsonl, finds by ID, transitions lifecycle from "new" to "acknowledged". Rewrites the file (read all, update target, write all back). Returns updated record or null if not found.

    3. `resolveWarning(warningId)` — Transitions lifecycle to "resolved", sets resolved_date to today. Same read-update-rewrite pattern. Returns updated record or null.

    4. `markStale(thresholdDays = 30)` — Reads all warnings, finds any with lifecycle "new" or "acknowledged" that are older than thresholdDays. Transitions them to "stale". Returns array of stale-marked records.

    5. `queryWarnings({ lifecycle, category, severity } = {})` — Reads warnings.jsonl, filters by provided criteria. Returns array of matching WarningRecord objects.

    6. `getWarningStats()` — Returns summary: { total, byLifecycle: { new: N, acknowledged: N, resolved: N, stale: N }, bySeverity: { error: N, warning: N, info: N }, oldestUnresolved: dateString|null }

    **Important implementation details:**
    - Use the compiled JS from scripts/reviews/dist/ for appendRecord and readValidatedJsonl (these are TypeScript files compiled to JS). Import pattern: `require('../reviews/dist/write-jsonl')` and `require('../reviews/dist/read-jsonl')`.
    - For the WarningRecord schema import: `require('../reviews/dist/schemas/warning')`.
    - For rewriting the full file (acknowledge/resolve/stale), read all records, modify target, write all back using fs.writeFileSync (with isSafeToWrite check from symlink-guard). This is safe because warnings.jsonl will be small (< 1000 records).
    - Use try/catch for all file reads (no existsSync).
    - If warnings.jsonl doesn't exist yet, queryWarnings and getWarningStats return empty results, createWarning creates the file.
