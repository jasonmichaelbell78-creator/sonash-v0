  <action>
    **Create `scripts/reviews/lib/enforcement-manifest.ts`:**
    Define Zod schema for enforcement manifest records:
    ```typescript
    {
      pattern_id: string,           // From CODE_PATTERNS.md heading/slug
      pattern_name: string,         // Human-readable name
      priority: "critical" | "important" | "edge",
      category: string,             // Category from CODE_PATTERNS.md
      mechanisms: {
        regex: "active:{rule-id}" | "migrated:{target}" | "none",
        eslint: "active:{rule-name}" | "none",
        semgrep: "active:{rule-id}" | "none",
        cross_doc: "linked" | "none",
        hooks: "pre-commit" | "pre-push" | "none",
        ai: "claude-md" | "code-reviewer" | "none",
        manual: "code-review" | "documented-only" | "none"
      },
      coverage: "automated" | "ai-assisted" | "manual-only" | "none",
      status: "active" | "stale" | "deprecated",
      last_verified: string         // ISO date
    }
    ```
    Export the schema, type, and helper functions:
    - `classifyCoverage(mechanisms)`: Returns "automated" if any of regex/eslint/semgrep/hooks is active, "ai-assisted" if only ai is active, "manual-only" if only manual, "none" otherwise.
    - `isStale(record)`: Returns true if no mechanism is active (all "none") and status is not "deprecated".

    **Create `scripts/reviews/build-enforcement-manifest.ts`:**
    CLI that builds the manifest by scanning all sources:

    1. Parse CODE_PATTERNS.md to extract all pattern IDs and names (from headings and subheadings under each category). Use the existing heading structure: `### N. Pattern Name` under category headings.

    2. For each pattern, check enforcement across mechanisms:
       - **Regex**: Read check-pattern-compliance.js, extract active rule IDs. Match against pattern name/id using fuzzy matching (rule ID contains pattern slug).
       - **ESLint**: Read eslint-plugin-sonash/index.js, extract rule names. Match against pattern name.
       - **Semgrep**: Scan .semgrep/rules/**/*.yml, extract `metadata.code-pattern-ref` values. Direct match against pattern slug.
       - **Cross-doc**: Check if pattern is referenced in doc-dependencies.json.
       - **Hooks**: If regex or ESLint rule is active, hooks = "pre-commit" (since check-pattern-compliance and ESLint both run in pre-commit).
       - **AI**: If pattern appears in CLAUDE.md Section 4, ai = "claude-md".
       - **Manual**: Default to "code-review" for all patterns (human reviews always apply).

    3. Classify coverage and detect staleness for each pattern.

    4. Write to data/ecosystem-v2/enforcement-manifest.jsonl using appendRecord from Phase 1 utilities.

    5. Print summary:
       ```
       Enforcement Manifest Built:
       - Total patterns: N
       - Automated (regex/eslint/semgrep/hooks): X (Y%)
       - AI-assisted: A (B%)
       - Manual-only: M (N%)
       - No enforcement: Z (W%)
       - Stale patterns: S
       ```

    CLI usage: `npx ts-node scripts/reviews/build-enforcement-manifest.ts [--dry-run]`

    **Create `scripts/reviews/verify-enforcement-manifest.ts`:**
    CLI that cross-references the manifest against actual rule files to detect drift:

    1. For each manifest record with regex/eslint/semgrep = "active:{id}":
       - Verify the referenced rule ID actually exists in the corresponding rule file
       - Report missing rules (manifest says active but rule not found)
       - Report untracked rules (rule exists but not in manifest)

    2. Print verification report:
       ```
       Enforcement Manifest Verification:
       - Records checked: N
       - Valid references: X
       - Missing rules (drift): Y
       - Untracked rules: Z
       - PASS/FAIL
       ```

    3. Exit with code 1 if any drift detected (for CI integration).

    CLI usage: `npx ts-node scripts/reviews/verify-enforcement-manifest.ts`

    **Generate initial manifest:**
    Run the builder to create the initial `data/ecosystem-v2/enforcement-manifest.jsonl`.

  </action>
  <verify>
    Run: `npx ts-node scripts/reviews/build-enforcement-manifest.ts` -- completes without error.
    Run: `npx ts-node scripts/reviews/verify-enforcement-manifest.ts` -- PASS (no drift).
    Check: `wc -l data/ecosystem-v2/enforcement-manifest.jsonl` matches pattern count from CODE_PATTERNS.md.
    Check: Coverage output shows automated enforcement >= 55%.
  </verify>
  <done>
    Enforcement manifest exists with per-pattern status across 7 mechanisms. Builder scans all rule sources. Verifier detects drift. Initial manifest generated with accurate coverage calculation.
  </done>
